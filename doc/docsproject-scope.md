# Project Scope

## Project Title
Campus Coffee and Catering Services Web Application

## Purpose
The project aims to build a centralized campus web application for coffee ordering, food ordering, and catering services. The platform will improve efficiency for students, faculty, event organizers, and food providers by replacing fragmented ordering methods such as calls, messaging apps, and in-person visits.

## Target Users
- Students
- Faculty
- Event organizers
- Food vendors
- Administrators

## Core Objectives
- Build a digital ordering platform for campus coffee and food services
- Support individual orders and catering bookings in one system
- Provide vendor and administrator management tools
- Simulate multi-method payment processing
- Support user personalization such as dietary preferences
- Deliver a mobile-responsive web application

## MVP Features
The following features are considered the minimum viable product:

### 1. User Account System
- User registration
- User login
- JWT-based authentication
- User profile management

### 2. Menu Browsing
- View menu items
- Browse categories
- Check item information and availability

### 3. Ordering System
- Add items to cart
- Place individual orders
- View order details

### 4. Catering Booking
- Submit catering requests
- View catering package information
- Support approval workflow for bookings

### 5. Vendor/Admin Functions
- Vendor order management
- Basic admin oversight
- Management interfaces for menus and orders

## Planned Advanced Features
These features may be added after core functions are completed:
- Real-time order tracking
- Sales reporting analytics
- Feedback and rating system
- Multi-method payment simulation
- QR code ordering support
- Enhanced personalization and dietary recommendation features

## Out of Scope for Early Stage
The following are not required in Week 1:
- Full implementation of all backend APIs
- Complete payment integration
- Full dashboard implementation
- Deployment to production
- Final testing and optimization

## Main Data Entities
The system is expected to include these main collections:
- Users
- Menu Items
- Orders
- Catering Packages
- Vendors
- Feedback

## Technical Scope
- Frontend: React.js
- Backend: Node.js + Express.js
- Database: MongoDB + Mongoose
- API Style: RESTful API
- Authentication: JWT
- UI Prototype: Figma
- Version Control: GitHub

## Week 1-2 Scope
For Week 1-2, the team should focus on:
- Discussing requirements
- Finalizing project scope
- Setting up the GitHub repository
- Defining team responsibilities clearly
- Preparing initial documentation
- Creating UI prototypes in Figma
- Initializing frontend and backend project skeletons
