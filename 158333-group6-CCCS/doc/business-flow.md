# Business Flow

## Catering Booking Flow

1 User opens catering page  
2 User selects package  
3 User fills booking form  
4 User submits request  
5 System creates booking  
6 Booking status = pending  
7 Vendor or admin reviews request  
8 Booking is approved or rejected  
9 User checks booking result  

---

## Approval Workflow

Status:

pending  
approved  
rejected  
completed  
cancelled  

Flow:

pending → approved  
pending → rejected  
approved → completed  
pending → cancelled  

---

## Vendor Flow

1 Vendor logs in  
2 Vendor views orders and bookings  
3 Vendor reviews booking requests  
4 Vendor updates booking status  
5 Vendor manages menu and services
